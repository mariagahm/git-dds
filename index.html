Perfect! Now you're probably wondering how to get your teammate's changes on your computer. Let the sharing begin!

Follow these steps:

  1. Open your github app.
  2. Notice that the index.html file has some changes to it: the changes you just made!
  3. Type in a commit message to describe what update you are making. Example: I updated my bio on the page!
  4. Press commit.
  5. Refresh the webpage.

Hmmm...nothing changed; Your teammate's stuff is not there, and if you check your repo on github.com, your commit is not there! The reason is because you haven't synced your changes to github yet.

To make it work, press on the sync button. Refresh the page, and you should see your teammate's changes.

Exercise 3

I should be red and blinking! If I'm not, then something is wrong.

Admire the text above. It's beautiful, isn't it?

Javascript is what's making the text blink. You can see the code inside index.html. Can you find it?

CSS is what's making the text red. It is also in index.html. Can you find it?

Sometimes it's good to have your css and javascript in the same files as your html; normally it's when you're just starting a project. But quickly your CSS and javascript will become big, and it will create a MASSIVE html file that is pretty confusing.

That's why it's best to split them into their own files.

Team Member 1

  1. Take all the CSS and put it in main.css inside the css directory. Don't forget to save your index.html.
  2. Refresh your page. Make sure the text is still red. If it's not, you need to link the main.css file to your html.
  3. Commit & sync your changes to github.

Team Member 2

  1. Take all the javascript and put it in main.js inside the js directory. Don't forget to save your index.html.
  2. Refresh your page. Make sure the text is still blinking. If it's not, you need to link the main.js file to your html.
  3. Commit & sync your changes to github.

Both of your projects should now be in sync. The javascript & CSS should be out of your index.html.

You make a great team!
